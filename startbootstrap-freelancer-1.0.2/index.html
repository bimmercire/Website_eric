    Personal Projects

    As an electrical engineer, I do like to play around with little projects, especially with an embedded environment. As a student, I was more focused with the hardware side of electrical engineering (Op Amps, Analog Electronics). With my subtle transition to a more embedded systems environment, I had to really practice my programming and micro-controller peripherals. Thus, here are some projects I have been working on.

    AeroBerry

    AeroBerry, an autonomous Infrared Tracking Quadcopter, was a senior design project I was apart of. My role in the project was more on the hardware side as I dealt with researching and constructing the Quadcopter, researching the power budget for all the components, designing a power system that would not only power all the different components, but also sustain the weight and flight time, and learning how to pilot.

    Responsibilities:

  • We had to decide upon a quadcopter suited for our tasks and our limited timeline. We decided on an DIY quadcopter kit from AeroQuad, supplying all the necessary parts for construction, including a flight controller board. I was tasked to research and assemble the configuration of the quadcopter needed to handle the weight of our payload.

  • I researched the power budget needed to not only power the quadcopter, but multiple microcontrollers, sensors. The quadcopter was to been powered by lithium polymer batteries, which I had no experience with, so I had to research which type I would need, how to charge it, and any saftey concerns I need to take account with. Once I completed the power budget, I designed the power system using switch mode power converters to safely step down voltages required by the microcontrollers and sensors.

  • Once construction and assembly was finished, we also had to learn how to pilot and correctly calibrate the quadcopter. As we were learning more about flying and calibrating, we moved into the next phase of autonomous flight, which I assisted in designing.

    QR Decryption Card

    The QR scanner/decoder decryption card was designed to scan encrypted QR codes (served by a host) and use Sunstone's decryption algorithm to solve and display the answer. The QR decryption card is unique to each device with its only decryption key associated with it. Given a paired encryption key, the host who serves the QR codes, would successfully encrypt its content having only the one unique QR decryption card successfully decrypt.

    Tasks:

  • First task was to find an open source QR library. Zbar bar code reader is an open source software that reads bar codes from various sources. I chose Zbar because of its small memory footprint, embedded capabilities, and small code size.

  • I was also researching microcontrollers and found a development board by Olimex that suited our requirements. The development board housed the microcontroller (STM32F407), a CMOS camera, GLCD display, and many more components.

  • I designed and implemented the decryption algorithm that would, given the device's unique private key, decrypt the host-encrypted QR code.

  • I took the schematic of the development board, using only the components I needed and designed the card using Eagle CAD and sent it off the manufacture. Since Olimex designed the boards, we used them to layout the boards.

    Private Secure Messaging App

    The private secure messaging app is designed to send encrypted messages between two contacts who wish to keep their conversation private. Given two contacts, each person with an phone and a watch (whose connected to the phone), registers a link between each other, generating an encryption and decryption key between those two contacts, thus allowing private messaging within the app.

    Tasks:

  • I was tasked to design the android app that would act as an private messaging app. I designed the layout of how a user would interact with the app. I designed the Login, Message list, Contact List, New Message, New Contact activities that a typical messaging application would require.

  • The app is designed the communicate with the Pebble watch using bluetooth. We used Pebble's open source API to establish, send, and receive data with an Android phone.

  • The app is also designed to communicate with a server, so we constructed a server that hosted a Login and Register Account page that would allow users to edit and save their contact info. The server also includes a RSA encryption/decryption key generator.

  • The Pebble Watch has to hold big integers for its encrypting and decrypting methods, so we constructed a big integer library.

    Detecting Malicious Web Activity

    I was tasked to create web traps using JavaScript and CSS to detect any malicious web activity. Web traps include detecting password managers, automated web scripts, text-based browsers.

    Tasks:

  • I learned how to design and develop web pages for prospective clients (mainly banking sites) who may need extra-ended security on login pages. Taking their sites as templates, I came up with a variety of traps using a combination of page manipulation, JavaScript (JQuery), and CSS.

  • I also learned how to serve my traps onto the site using Nginx. Within Nginx, I had to devise a way to check if the user tripped those traps and determine if they were a regular human, or malicious activity.
